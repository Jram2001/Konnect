06ai prompt contract
Claude receives:
- article title + body

Claude must return JSON array:
[{ entity_key, display_name, event_text, event_category, impact_score (-5 to +5), reasoning, is_macro }]

entity_key rules:
- lowercase, format: name_type (e.g. "tesla_company", "nikola_tesla_person")
- naming convention is self-normalizing — no registry list needed in prompt
- "Tesla Inc", "TSLA", "Tesla Motors" all → "tesla_company"
- "Nikola Tesla" → "nikola_tesla_person" (different entity)
- never vary the key across articles

is_macro = true when the event affects an entire sector/industry, not just one entity
07daily pipeline flow
06:00 AM
1. fetch news from APIs
2. dedup by url_hash, store articles
3. for each unprocessed article:
3a. send to claude (no registry list needed — entity_key format is self-normalizing)
3b. for each extraction: upsert entity by entity_key
3c. bulk insert connections
3d. mark article processed
4. for each active user:
4a. collect entity_keys from watchlist
4b. lookup macro_group_keys from entities
4c. query connections: entity_key $in watchlist_keys (micro)
4d. query connections: macro_group_keys $in user_groups, is_macro=true
4e. merge + rank by abs(impact_score)
4f. claude generates mood + subject line
4g. create digest doc
5. for each pending digest:
5a. render email html
5b. send via sendgrid
5c. mark delivered
